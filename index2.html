Táblákban alkalmazható korlátozások bemutatása

PRIMARY KEY

A PRIMARY KEY korlátozás egyedi azonosítóként szolgál egy táblában, és nem engedélyezi a NULL értékeket.

CREATE TABLE tabla_neve (
    id INT PRIMARY KEY,
    nev VARCHAR(50)
);

FOREIGN KEY

A FOREIGN KEY korlátozás egy táblában egy másik tábla oszlopára hivatkozik, és biztosítja az adatok integritását.

CREATE TABLE tabla_neve (
    id INT PRIMARY KEY,
    nev VARCHAR(50),
    masik_id INT,
    FOREIGN KEY (masik_id) REFERENCES masik_tabla(id)
);

UNIQUE

A UNIQUE korlátozás biztosítja, hogy egy oszlopban szereplő értékek egyediek legyenek.

CREATE TABLE tabla_neve (
    id INT PRIMARY KEY,
    nev VARCHAR(50) UNIQUE
);

NOT NULL

A NOT NULL korlátozás megköveteli, hogy egy oszlopban ne lehessen NULL érték.

CREATE TABLE tabla_neve (
    id INT PRIMARY KEY,
    nev VARCHAR(50) NOT NULL
);

DEFAULT

A DEFAULT korlátozás megad egy alapértelmezett értéket egy oszlop számára, ha nem adunk meg értéket az adatbevitel során.

CREATE TABLE tabla_neve (
    id INT PRIMARY KEY,
    nev VARCHAR(50) DEFAULT 'N/A'
);

CHECK

A CHECK korlátozás megad egy feltételt, amelyet az oszlop értékeinek teljesíteniük kell.

CREATE TABLE tabla_neve (
    id INT PRIMARY KEY,
    nev VARCHAR(50),
    kor INT CHECK (kor >= 0)
);
